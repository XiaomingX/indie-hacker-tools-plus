# awesome-actions
## 精选实用的GitHub Actions清单（含官方核心工具与社区热门方案）
GitHub Actions 是 GitHub 内置的自动化工具，可帮你实现代码构建、测试、部署、issue 管理等几乎所有开发流程的自动化。这份清单整理了**官方核心工具**与**社区高价值方案**，帮你快速找到适合的自动化工具。


## 一、官方核心资源
这部分是 GitHub 官方提供的基础资源，新手入门或查找权威信息首选。

- **官方站点**：[GitHub Actions 首页](https://github.com/features/actions)  
  快速了解 Actions 核心能力、应用场景和入门案例。
- **官方文档**：[GitHub Actions 文档](https://docs.github.com/en/actions)  
  权威的详细教程（含语法、运行环境、API 等），原旧链接已更新。
- **官方 Actions 组织**：[github/actions](https://github.com/actions)  
  官方维护的核心工具仓库，包含：
  - [GitHub Actions 运行器](https://github.com/actions/runner)：Actions 自动化任务的执行引擎（本地自托管 runner 必看）。
  - [GitHub Actions 运行环境镜像](https://github.com/actions/runner-images)：官方提供的虚拟环境（如 Ubuntu、Windows、macOS）配置，**原 `virtual-environments` 仓库已废弃**。


## 二、核心官方Action（工作流必备）
官方维护的高频使用 Action，稳定性强，是大多数工作流的基础。

### 1. 工作流基础工具
这些是几乎所有工作流的“基础设施”，处理代码获取、文件传递等核心操作。
- [actions/checkout](https://github.com/actions/checkout)  
  核心用途：拉取当前代码仓库到 Actions 运行环境，**所有需要操作代码的工作流必加第一步**。
- [actions/upload-artifact](https://github.com/actions/upload-artifact)  
  核心用途：保存工作流中生成的文件（如构建产物、测试报告、日志），支持跨作业（Job）共享或手动下载。
- [actions/download-artifact](https://github.com/actions/download-artifact)  
  核心用途：下载由 `upload-artifact` 保存的文件（比如在“构建作业”后，“部署作业”下载产物）。
- [actions/cache](https://github.com/actions/cache)  
  核心用途：缓存依赖包（如 npm、pip、Maven）或构建产物，**避免每次运行都重新下载/构建，平均提速 50% 以上**。
- [actions/github-script](https://github.com/actions/github-script)  
  核心用途：在工作流中直接编写 JavaScript 代码调用 GitHub API（比如自动评论 issue、修改 PR 状态）。

### 2. GitHub 自动化工具
专门用于自动化 GitHub 平台内的操作（如发布、标签、issue 管理）。
- [softprops/action-gh-release](https://github.com/softprops/action-gh-release)  
  核心用途：创建 GitHub Release（发布版本）并上传附件（如安装包、二进制文件）。  
  说明：**官方旧版 `create-release` 和 `upload-release-asset` 已停止维护，此为社区首选替代方案**。
- [actions/stale](https://github.com/actions/stale)  
  核心用途：自动标记长期未更新的 issue/PR（比如 30 天无互动标为“stale”，再 7 天无反应自动关闭）。  
  说明：也可考虑社区增强版 [actions/stale-issues](https://github.com/actions/stale-issues)，支持更灵活的规则。
- [actions/labeler](https://github.com/actions/labeler)  
  核心用途：基于 PR 的文件变更路径自动打标签（比如改了 `src/pages/` 下的文件，自动加“前端”标签）。

### 3. 编程语言环境配置
一键搭建指定版本的编程语言运行环境，无需手动安装依赖。
- [actions/setup-node](https://github.com/actions/setup-node)：配置 Node.js 环境（支持指定 v16/v18/v20 等版本，自动缓存 npm 依赖）。
- [actions/setup-python](https://github.com/actions/setup-python)：配置 Python 环境（支持指定版本，可联动 pip 缓存）。
- [actions/setup-java](https://github.com/actions/setup-java)：配置 Java 环境（支持 JDK 8/11/17，适配 Maven/Gradle）。
- [actions/setup-go](https://github.com/actions/setup-go)：配置 Go 环境（官方维护，替代旧版社区方案，支持 Go Modules）。
- [actions/setup-dotnet](https://github.com/actions/setup-dotnet)：配置 .NET 环境（支持 .NET Core/.NET 5+ 等版本）。


## 三、自定义Action开发
如果官方/社区工具满足不了需求，可自己开发 Action，这些是开发必备工具。
- [actions/toolkit](https://github.com/actions/toolkit)  
  核心用途：GitHub 官方提供的 Action 开发框架（基于 JavaScript/TypeScript），封装了日志、授权、文件操作等常用能力。
- [actions/hello-world-javascript-action](https://github.com/actions/hello-world-javascript-action)  
  核心用途：最简单的 JavaScript Action 示例模板，新手入门自定义开发的“Hello World”。


## 四、社区明星工具（提升效率神器）
社区维护的高口碑工具，解决 Actions 使用中的核心痛点（如本地调试、标签同步）。
- [nektos/act](https://github.com/nektos/act)  
  核心用途：在本地终端运行 GitHub Actions 工作流，**无需推代码到 GitHub 即可调试，大幅节省开发时间**。
- [atrox/github-actions-badge](https://github.com/atrox/github-actions-badge)  
  核心用途：生成工作流运行状态的徽章（如“passed/failed”），可嵌入 README 展示自动化状态。
- [micnncim/action-label-syncer](https://github.com/micnncim/action-label-syncer)  
  核心用途：同步代码仓库的标签（比如从模板仓库同步“bug”“feature”等标签），统一多仓库标签规范。


## 五、按场景分类的实用Action
### 1. 依赖管理（提速+合规）
- [bahmutov/npm-install](https://github.com/bahmutov/npm-install)：智能缓存 npm 依赖，比 `npm install` 更快，支持锁定文件校验。
- [actions/setup-pnpm](https://github.com/actions/setup-pnpm)：配置 pnpm 包管理器环境，自动缓存依赖。
- [c-hive/gha-yarn-cache](https://github.com/c-hive/gha-yarn-cache)：专门缓存 Yarn 依赖，适配 Yarn 1/2+ 版本。
- [hiwelo/new-dependencies-action](https://github.com/hiwelo/new-dependencies-action)：在 PR 中高亮显示新增的 npm 依赖，方便审核第三方包风险。

### 2. 代码质量与测试
#### （1）静态分析与代码风格
- [OskarStark/phpstan-ga](https://github.com/OskarStark/phpstan-ga)：运行 PHPStan 进行 PHP 静态代码分析，暴露类型错误等问题。
- [reviewdog/action-eslint](https://github.com/reviewdog/action-eslint)：用 ESLint 检查 JavaScript/TypeScript 代码，直接在 PR 中显示错误（替代已废弃的 TSLint）。
- [peter-evans/autopep8](https://github.com/peter-evans/autopep8)：自动格式化 Python 代码，符合 PEP 8 规范。
- [kamilkisiela/graphql-inspector](https://github.com/kamilkisiela/graphql-inspector)：检查 GraphQL  schema 变更是否兼容，避免接口break。

#### （2）测试执行
- [cypress-io/github-action](https://github.com/cypress-io/github-action)：运行 Cypress 进行前端 E2E 测试（支持缓存测试环境，生成测试报告）。
- [DevExpress/testcafe-action](https://github.com/DevExpress/testcafe-action)：运行 TestCafe 测试（无需配置浏览器驱动，支持多浏览器）。
- [ianwalter/puppeteer](https://github.com/ianwalter/puppeteer)：基于 Puppeteer 运行前端自动化测试（如页面截图、交互模拟）。

#### （3）代码覆盖率
- [codecov/codecov-action](https://github.com/codecov/codecov-action)：将测试覆盖率数据上传到 Codecov 平台，生成可视化报告。
- [sonarsource/sonarcloud-github-action](https://github.com/sonarsource/sonarcloud-github-action)：用 SonarCloud 扫描代码（覆盖覆盖率、漏洞、代码异味等多维度质量）。

### 3. PR与Issue管理
- [pullreminders/assignee-to-reviewer-action](https://github.com/pullreminders/assignee-to-reviewer-action)：自动将 PR 负责人设为审核人，简化审核流程。
- [pascalgn/automerge-action](https://github.com/pascalgn/automerge-action)：满足条件时自动合并 PR（如“所有审核通过+测试通过”）。
- [banyan/auto-label](https://github.com/banyan/auto-label)：根据 PR 标题/描述自动打标签（比如含“fix”加“bug”标签）。

### 4. 版本管理与发布
- [WyriHaximus/github-action-next-semvers](https://github.com/WyriHaximus/github-action-next-semvers)：根据语义化版本规则（SemVer）自动生成下一个版本号（如 v1.2.3 → v1.2.4）。
- [christian-draeger/increment-semantic-version](https://github.com/christian-draeger/increment-semantic-version)：手动指定版本增量（major/minor/patch），更新版本号文件。

### 5. 部署发布
#### （1）静态网站部署
- [peaceiris/actions-gh-pages](https://github.com/peaceiris/actions-gh-pages)：部署静态网站到 GitHub Pages（支持 Hugo、Vue、React 等框架，适配自定义域名）。
- [peaceiris/actions-hugo](https://github.com/peaceiris/actions-hugo)：配置 Hugo 环境并构建，配合上一个 Action 部署 Hugo 站点。

#### （2）容器化部署
- [docker/build-push-action](https://github.com/docker/build-push-action)：构建 Docker 镜像并推送到 Docker Hub、GitHub Container Registry 等仓库（官方推荐，功能全面）。
- [appleboy/docker-ecr-action](https://github.com/appleboy/docker-ecr-action)：推送 Docker 镜像到 AWS ECR 仓库。

#### （3）云服务部署
- [jakejarvis/s3-sync-action](https://github.com/jakejarvis/s3-sync-action)：同步文件到 AWS S3 存储（常用于静态资源托管）。
- [netlify/actions](https://github.com/netlify/actions)：部署静态网站到 Netlify（支持自动预览 PR 变更）。
- [steebchen/kubectl](https://github.com/steebchen/kubectl)：通过 kubectl 部署应用到 Kubernetes 集群。

### 6. 通知与告警
- [Ilshidur/action-discord](https://github.com/Ilshidur/action-discord)：工作流状态变更时发送通知到 Discord 频道（支持自定义消息格式）。
- [actions/slack-notify](https://github.com/actions/slack-notify)：发送通知到 Slack（比旧版 `slack-action` 更轻量、配置更简单）。
- [appleboy/telegram-action](https://github.com/appleboy/telegram-action)：发送消息到 Telegram 机器人（支持 Markdown 格式）。
- [peter-evans/sendgrid-action](https://github.com/peter-evans/sendgrid-action)：通过 SendGrid 发送邮件通知（如部署成功提醒）。

### 7. 安全与合规
- [aquasecurity/trivy-action](https://github.com/aquasecurity/trivy-action)：扫描 Docker 镜像或代码中的漏洞（社区主流工具，支持 CVE 漏洞库）。
- [ridedott/dependabot-auto-merge-action](https://github.com/ridedott/dependabot-auto-merge-action)：自动合并 Dependabot 提出的依赖更新 PR（需满足测试通过等条件）。

### 8. 环境配置
- [SpicyPizza/create-envfile](https://github.com/SpicyPizza/create-envfile)：根据 Secrets 动态生成 `.env` 文件（避免硬编码环境变量）。
- [goanpeca/setup-miniconda](https://github.com/goanpeca/setup-miniconda)：配置 Conda 环境（常用于 Python 数据科学项目）。
