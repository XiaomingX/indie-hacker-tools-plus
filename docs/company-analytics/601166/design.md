# 兴业银行（601166.SH）深度研究技术设计方案

## 1. 研究框架设计

### 1.1 分析模型选择

| 维度 | 分析模型/方法 | 说明 |
|------|--------------|------|
| 宏观分析 | 美林时钟 + 政策Beta框架 | 定位利率周期与政策导向 |
| 行业分析 | 波特五力 + 银行股特色指标 | NIM、资产质量、资本充足率 |
| 商业模式 | 营收拆解 + 护城河评估 | 杜邦分析法拆解ROE |
| 竞争格局 | CR5集中度 + 差异化矩阵 | 股份行横向对比 |
| 财务质量 | 审计底稿思维 + FCF分析 | 交叉验证利润质量 |
| 估值分析 | PB-ROE + DDM + 历史分位 | 多维度估值交叉验证 |
| 风险分析 | Pre-mortem + 压力测试 | 极端场景推演 |
| 情绪资金 | 机构持仓 + 北向资金 + 融资融券 | 市场微观结构 |

### 1.2 银行业特殊指标体系

```
核心指标：
├── 盈利能力
│   ├── 净息差（NIM）= 净利息收入 / 生息资产平均余额
│   ├── ROE = 归母净利润 / 归母净资产
│   ├── ROA = 归母净利润 / 总资产平均余额
│   └── 成本收入比 = 业务及管理费 / 营业收入
├── 资产质量
│   ├── 不良贷款率 = 不良贷款 / 贷款总额
│   ├── 拨备覆盖率 = 贷款减值准备 / 不良贷款
│   ├── 关注类贷款迁徙率
│   └── 逾期90天以上/不良贷款（偏离度）
├── 资本实力
│   ├── 核心一级资本充足率
│   ├── 总资本充足率
│   └── TLAC达标比率
└── 估值指标
    ├── PB（市净率）= 股价 / 每股净资产
    ├── 股息率 = 每股分红 / 股价
    └── PB-ROE散点图定位
```

## 2. 数据采集方案

### 2.1 数据源优先级

1. **一手数据**：公司公告、年报/季报原始数据
2. **监管数据**：央行货币政策报告、银保监会统计数据
3. **终端数据**：Wind/Choice金融终端（最优先使用MCP工具获取）
4. **研报数据**：卖方研报（强制打折20%处理）

### 2.2 关键时间节点

| 数据类型 | 截止时间要求 |
|----------|-------------|
| 年报数据 | 2025年报（最新完整年度） |
| 季报数据 | 2026年一季报（如有） |
| 估值数据 | 实时最新 |
| 政策数据 | 最新监管文件 |

## 3. 输出规范

### 3.1 报告文件结构

```
reports/601166/
├── requirements.md          # 本文件 - 需求文档
├── design.md                # 本文件 - 设计方案
├── tasks.md                 # 任务分解
├── 1_macro_policy_industry.md   # 宏观政策行业
├── 2_business_model.md          # 商业模式
├── 3_competitor_analysis.md     # 竞争格局
├── 4_financial_quality.md       # 财务质量
├── 5_governance.md              # 治理结构
├── 6_valuation_safety.md        # 估值安全边际
├── 7_catalysts.md               # 催化剂
├── 8_reverse_analysis.md        # 逆向分析
├── 9_sentiment_funds.md         # 资金情绪
├── 10_munger_checklist.md       # 芒格检查清单
└── 11_final_report.md           # 最终报告
```

### 3.2 格式规范

- 每个报告文件开头包含：报告标题、分析日期、数据截止时间
- 关键数据用表格呈现，便于横向对比
- 核心结论用加粗或引用块突出
- 所有数字保留合理精度（百分比保留2位小数）

## 4. 交互流程设计

```
Step 1: 宏观政策行业 → 暂停确认 → Step 2
Step 2: 商业模式 → 暂停确认 → Step 3
Step 3: 竞争格局 → 暂停确认 → Step 4
Step 4: 财务质量 → 暂停确认 → Step 5
Step 5: 治理结构 → 暂停确认 → Step 6
Step 6: 估值安全边际 → 暂停确认 → Step 7
Step 7: 催化剂 → 暂停确认 → Step 8
Step 8: 逆向分析 → 暂停确认 → Step 9
Step 9: 资金情绪 → 暂停确认 → Step 10
Step 10: 芒格检查 → 暂停确认 → Step 11
Step 11: 最终报告 → 完成
```

每个步骤完成后暂停，询问用户：
- 数据是否准确？
- 分析逻辑是否符合预期？
- 是否有最新边际变化需要补充？
