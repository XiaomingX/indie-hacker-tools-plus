# MiniMax - 投研分析设计方案

## 1. 核心分析主线

```
宏观政策 → 行业格局 → 商业模式 → 财务健康 → 治理结构 → 估值判断 → 催化剂 → 风险排雷 → 终极决策
```

## 2. 关键假设与检验点

| 假设编号 | 假设内容 | 检验方法 |
|---------|---------|---------|
| H1 | MiniMax的C端社交产品（Talkie/星野）具备可持续的用户粘性 | DAU/MAU、留存率、付费率分析 |
| H2 | 多模态能力（视频+语音+文本）构成差异化护城河 | 技术Benchmark、产品体验、竞品对比 |
| H3 | 商业模式已验证（订阅+广告+API） | 营收结构、UE模型、毛利率分析 |
| H4 | 版权诉讼和数据蒸馏指控不影响核心业务 | 诉讼进展、财务影响评估 |
| H5 | 与米哈游的早期关系带来游戏/娱乐场景协同 | 合作深度、收入贡献 |

## 3. 分析模块设计

### Module 1: 宏观与行业
- 与智谱AI报告共享行业背景
- 重点关注C端AI应用的市场空间
- 社交娱乐赛道的政策风险（内容审核）

### Module 2: 商业模式
**核心问题**：
- C端社交产品的变现模式是否可持续？
- Talkie 1100万MAU的含金量如何？
- 订阅、广告、API三条腿走路的结构是否健康？
- 与Character.ai的商业模式对比

### Module 3: 竞争格局
**对标公司**：
- C端：Character.ai、Replika、Poe
- 多模态：OpenAI（Sora）、Runway、Pika
- 国内同行：智谱AI、月之暗面、DeepSeek

### Module 4: 财务质量
**特殊关注**：
- C端产品的获客成本（CAC）和用户生命周期价值（LTV）
- 广告收入占比和ARPU
- 版权诉讼的潜在财务影响

### Module 5: 治理结构
**特殊关注**：
- 商汤系创始团队的背景和能力
- 米哈游早期投资的战略意义
- 腾讯、阿里的利益冲突

### Module 6: 估值
**对标方法**：
- 社交产品对标：Snap、Pinterest
- AI公司对标：Character.ai、OpenAI
- 混合估值方法

### Module 7-11: 催化剂、风险、情绪、检查、决策
- 重点关注版权诉讼风险
- 数据蒸馏指控的影响
- C端用户增长的可持续性

## 4. 质量控制

- 交叉验证：关键结论至少2个独立信源
- 证伪机制：每个正面结论附带反面论点
- 数据新鲜度：优先2025H2-2026Q1数据
- 可追溯性：所有数据点标注来源
