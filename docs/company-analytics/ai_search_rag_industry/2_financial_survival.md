# 2. 财务生存能力与单体模型对决

## 谁在变现“API”？谁在变现“流量”？

### 1. Perplexity：沉重的“双重收税”
- **单体模型**：Perplexity 每回答一个问题，都要给搜索引擎（如 Google/Bing）付检索费，还要给大模型厂商（如 OpenAI/Anthropic）付推理费。
- **财务特征**：**“负毛利”的风险极大**。除非其自研模型能承载 80% 以上的流量，否则它只是在帮巨头打工。
- **生存现状**：依赖于持续的大额融资来维持用户增长。

### 2. Tavily：高效的“中间件”
- **单体模型**：按 API 调用收费。由于其不涉及庞大的模型研发，其边际成本极低。
- **财务特征**：**极佳的现金流潜力**。只要 Agent 行业在增长，它的收入就是指数级的。
- **生存现状**：稳健。

### 3. Exa：早期的“技术积累期”
- **单体模型**：类似于 Tavily，但需要维护更庞大的语义索引库，算力开支高于 Tavily。
- **生存现状**：稳健。

## 财务指标对比 (2025 预估)

| 指标 | Perplexity | Tavily | Exa |
| :--- | :--- | :--- | :--- |
| **毛利率** | ~30%-40% (受限) | **~70%-80%** | ~60% |
| **单次调用成本** | **最高** | 极低 | 低 |
| **现金/烧钱比 (Runway)** | 取决于融资速度 | 良好 | 良好 |

## 结论：谁的财务逻辑更具韧性？

- **Tavily** 的逻辑最性感。它是 AI 时代的“中间商”，不承担模型研发的风险，却能收割流量爆发的红利。
- **Perplexity** 的逻辑最脆弱。它在两头受气，必须通过极大的规模效应来摊薄成本，或者彻底完成“搜索-模型”的垂直一体化。
