# 3. 核心企业业务参数与生存现状对决

## 三种流派的生死分野
Poki、CrazyGames 和 TapTap 代表了游戏分发赛道的三种生存哲学。

### 1. Poki：极致的“人工精选”B2C 平台
- **业务参数**：MAU 约 1 亿。采用类似 App Store 的严格审核机制。
- **生存策略**：通过帮助开发者优化 H5 游戏的 UI/UX 和加载速度，提升留存率。它不追求游戏数量，追求**“用户在网页上停留的时间”**。
- **定性**：它是 Web 端的“精品店”，护城河在于与顶级 H5 开发者（如《地铁跑酷》网页版）的长期分成协议。

### 2. CrazyGames：规模化的“超市”分发
- **业务参数**：极高的游戏更新频率。更强调开发者自助上传和广告自动化优化。
- **生存策略**：依靠庞大的游戏库覆盖几乎所有的长尾关键词。
- **定性**：它是 Web 端的“平价超市”，护城河在于极高的 SEO 覆盖率和自动化变现系统。在 AI 生成游戏泛滥的时代，其面临的垃圾内容过滤压力巨大。

### 3. TapTap：内容驱动的“硬核玩家社区”
- **业务参数**：2025 年心动公司净利润大幅增长，主要由自研爆款驱动。
- **生存策略**：**“以研带运”**。TapTap 不仅仅是商店，它必须自己下场做爆款，用爆款游戏把用户“绑架”到平台上，再利用平台收割其他厂商的广告费。
- **定性**：它是移动端的“Steam + 社区”。护城河在于玩家的信任背书（TapTap 分）。一旦“评分信誉”被广告污染，其根基就会动摇。

## 总结对比表 (2025 视角)
| 指标 | Poki | CrazyGames | TapTap (心动) |
| :--- | :--- | :--- | :--- |
| **变现核心** | 广告分成 (H5) | 广告套利 (H5) | 广告费 (移动原生) |
| **开发者关系** | 深度扶持与优化 | 开放式自助接入 | 0 分成协议 |
| **护城河** | 精选内容与用户留存 | 流量规模与 SEO | 社区评分与自研内容 |
| **2025 关键点** | 防御 AI 垃圾内容 | 规模化套利效率 | 自研产品能否持续爆款 |

## 最终结论
Poki 和 CrazyGames 是在收割“碎片化时间”，而 TapTap 是在收割“沉浸式玩家”。从商业弹性看，TapTap 的上限极高（可以转型为研发巨头），但 Poki 的底线更稳。在 AI 时代，Poki 这种拥有“人工审美过滤”的平台反而可能获得更高的溢价。
