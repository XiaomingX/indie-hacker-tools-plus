# 中国建筑（601668.SH）深度研究技术设计方案

## 1. 研究框架设计

### 1.1 分析模型选择

| 维度 | 分析模型/方法 | 说明 |
|------|--------------|------|
| 宏观分析 | 基建投资周期 + 政策Beta框架 | 逆周期调节与专项债驱动 |
| 行业分析 | 波特五力 + 建筑行业特色指标 | 订单转化率、收现比、在手合同 |
| 商业模式 | 营收拆解 + 建筑+地产协同评估 | 双轮驱动模式拆解 |
| 竞争格局 | 央企CR6对比 + 差异化矩阵 | 建筑央企横向比较 |
| 财务质量 | 审计底稿思维 + FCF/现金流分析 | 应收款质量、现金回款率 |
| 估值分析 | PE/PB分位 + DCF + 股息贴现 | 多维度估值交叉验证 |
| 风险分析 | Pre-mortem + 压力测试 | 地产敞口、应收款减值 |
| 情绪资金 | 机构持仓 + 北向资金 + 融资融券 | 市场微观结构 |

### 1.2 建筑行业特殊指标体系

```
核心指标：
├── 订单与收入
│   ├── 新签合同额增速 = (本期新签 - 上期新签) / 上期新签
│   ├── 在手合同保障倍数 = 在手合同额 / 年收入
│   ├── 订单转化率 = 确认收入 / 新签合同（滞后分析）
│   └── 基建/房建/地产新签占比
├── 盈利能力
│   ├── 综合毛利率 = 毛利 / 营业收入
│   ├── 净利率 = 归母净利润 / 营业收入
│   ├── ROE = 归母净利润 / 归母净资产
│   └── 各业务板块毛利率
├── 现金流质量
│   ├── 收现比 = 销售收到现金 / 营业收入
│   ├── 经营现金流 / 净利润
│   ├── 应收账款周转天数
│   └── 合同资产/合同负债变动
├── 资产质量
│   ├── 应收账款/营业收入
│   ├── 存货（地产开发成本）占比
│   ├── 有息负债率
│   └── 商誉/总资产
└── 估值指标
    ├── PE（市盈率）历史分位数
    ├── PB（市净率）历史分位数
    ├── 股息率 = 每股分红 / 股价
    └── 央企估值对标
```

## 2. 数据采集方案

### 2.1 数据源优先级

1. **一手数据**：公司公告、年报/季报原始数据
2. **监管数据**：住建部、国家统计局基建投资数据
3. **终端数据**：Wind/Choice金融终端（最优先使用MCP工具获取）
4. **研报数据**：卖方研报（强制打折20%处理）

### 2.2 关键时间节点

| 数据类型 | 截止时间要求 |
|----------|-------------|
| 年报数据 | 2025年报（最新完整年度） |
| 季报数据 | 2026年一季报（如有） |
| 估值数据 | 实时最新 |
| 政策数据 | 最新基建/地产政策文件 |

## 3. 输出规范

### 3.1 报告文件结构

```
reports/601668/
├── requirements.md          # 本文件 - 需求文档
├── design.md                # 本文件 - 设计方案
├── tasks.md                 # 任务分解
├── 1_macro_policy_industry.md   # 宏观政策行业
├── 2_business_model.md          # 商业模式
├── 3_competitor_analysis.md     # 竞争格局
├── 4_financial_quality.md       # 财务质量
├── 5_governance.md              # 治理结构
├── 6_valuation_safety.md        # 估值安全边际
├── 7_catalysts.md               # 催化剂
├── 8_reverse_analysis.md        # 逆向分析
├── 9_sentiment_funds.md         # 资金情绪
├── 10_munger_checklist.md       # 芒格检查清单
└── 11_final_report.md           # 最终报告
```

### 3.2 格式规范

- 每个报告文件开头包含：报告标题、分析日期、数据截止时间
- 关键数据用表格呈现，便于横向对比
- 核心结论用加粗或引用块突出
- 所有数字保留合理精度（百分比保留2位小数）

## 4. 交互流程设计

```
Step 1: 宏观政策行业 → 暂停确认 → Step 2
Step 2: 商业模式 → 暂停确认 → Step 3
Step 3: 竞争格局 → 暂停确认 → Step 4
Step 4: 财务质量 → 暂停确认 → Step 5
Step 5: 治理结构 → 暂停确认 → Step 6
Step 6: 估值安全边际 → 暂停确认 → Step 7
Step 7: 催化剂 → 暂停确认 → Step 8
Step 8: 逆向分析 → 暂停确认 → Step 9
Step 9: 资金情绪 → 暂停确认 → Step 10
Step 10: 芒格检查 → 暂停确认 → Step 11
Step 11: 最终报告 → 完成
```

每个步骤完成后暂停，询问用户：
- 数据是否准确？
- 分析逻辑是否符合预期？
- 是否有最新边际变化需要补充？
