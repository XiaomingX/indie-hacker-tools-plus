# 8. 极限压力测试 (Pre-mortem)

## 负面场景推演与资产风险敞口分析

### 测试场景 1：算力硬件底层架构技术变更
- **情景假设**：芯片能效比获得跨量级突破，或者非 GPU 架构（如光子芯片、低功耗 ASIC）成为主流，单卡功耗大幅下降。
- **风险敞口**：重资产投建的超高密液冷数据中心（因配置昂贵的冷板及专用管线）将面临严重的产能不匹配及闲置，相关的沉没成本将拉低全周期的资本回报率。

### 测试场景 2：算力租赁终端需求衰退与价格腰斩
- **情景假设**：受宏观资金紧缩影响，大模型行业融资受阻，对外算力采购急剧缩减，叠加云巨头价格战导致散租价格跌幅超 50%。
- **风险敞口**：财务杠杆较高的独立算力租赁企业将无法通过租金现金流入覆盖短期贷款利息与设备折旧。由于底层算力卡属于高折价流动资产，此类企业存在明显的清算破产风险。

### 测试场景 3：主要大客户转向自建计算集群
- **情景假设**：互联网云巨头出于数据安全和成本考量，逐步削减外部 IDC 租赁份额，全面转为自购土地、自建数据中心。
- **风险敞口**：客户集中度超 50% 且以互联网企业为主的第三方数据中心，在长约到期后将面临上架率断崖式下跌，且很难在短时间内找到体量相当的客户填补空缺。
