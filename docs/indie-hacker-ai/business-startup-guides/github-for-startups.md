# GitHub for Startups 扶持计划 (2026 Checklist)

> [!IMPORTANT]
> **Indie Hacker Insight**: 2026 年 GitHub 不再只是代码托管，更是 **"AI-Powered Developer Platform"**。
> - **Copilot 核心**：初创企业可以免费或极低成本获得 Copilot 授权，极大提升 1-2 人团队的产出效率。
> - **安全合规**：Advanced Security 帮助独立开发者在早期就建立起企业级的代码安全防线。

- [ ] [**GitHub Enterprise 免费席位**](https://github.com/enterprise/startups) - 第一年提供 **20 个免费席位**，第二年续费享 50% 折扣。

- [ ] [**GitHub Copilot 专项支持**](https://github.com/features/copilot) - 获得针对企业级 Copilot 的集成指导与最佳实践分享。
- [ ] [**专属技术咨询**](mailto:startups@github.com) - 直接联系 GitHub 技术专家进行架构评审。

## ✅ 申请资格与流程 (Eligibility & Application)

- [ ] **融资轮次**：处于 B 轮或更早阶段。
- [ ] **新用户要求**：此前未曾使用过 GitHub Enterprise 或未获得过相关信用额度。
- [ ] **合作伙伴关联**：需通过指定的合作伙伴（如 VC, 加速器）页面申请（[示例链接](https://resources.github.com/startups/?pid=EXAMPLE)）。
- [ ] **响应速度**：提交申请后，GitHub 团队通常在 1-2 个工作日内回复。

## 💡 开发者建议
1. **尽早申请**：即便只有 1-2 个人，Enterprise 版的权限管理和 Actions 配额也非常有用。
2. **利用 Copilot 提效**：在团队扩张前，利用 AI 编程助手完成 80% 的重复性代码工作。
3. **关注安全合规**：利用 Advanced Security 自动化扫描依赖漏洞，降低后续运维风险。
4. **联系邮箱**：如有任何疑问，可直接联系 [startups@github.com](mailto:startups@github.com)。
