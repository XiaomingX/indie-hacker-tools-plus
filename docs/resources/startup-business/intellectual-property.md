# 2026 独立开发者知识产权 (IP) 防御指南 (Checklist)

> [!IMPORTANT]
> **Indie Hacker Insight**: 2026 年，知识产权不再只是大公司的法律武器，而是独立开发者保护核心资产（尤其是代码与模型权重）的关键。
> - **开源协议选型**：明确你的项目是想走商业化（推荐 **AGPL-3.0** 或 **BSL**）还是纯公益（**MIT/Apache**）。
> - **模型 IP**：如果你训练了微调模型，确保训练数据的版权清晰，以避免潜在的法律诉讼。
> - **防御性注册**：核心域名和主要市场的商标应尽早注册，防止被恶意抢注。

---

## 🛡️ 核心权利分类 (Core IP Rights)

- [ ] [**著作权 (Copyright)**](https://www.copyright.gov/) - 自动保护你的源代码、UI 设计、音乐与文档。2026 年应特别关注 AI 生成内容的版权归属。
- [ ] [**商标权 (Trademarks)**](https://www.uspto.gov/trademarks) - 保护你的产品名称（如 "Cursor"）与 Logo，防止混淆。
- [ ] [**专利权 (Patents)**](https://www.uspto.gov/patents) - 针对独特的技术发明（如一种新型的检索算法），通常成本较高，适合已获得融资的项目。
- [ ] [**商业秘密 (Trade Secrets)**](https://www.wipo.int/tradesecrets/en/) - 保护不公开的技术细节、客户名单或算法权重，通过 NDA (保密协议) 维持。

---

## ✅ 独立开发者实战建议 (Action Items)

- [ ] **代码库协议管理**：在 GitHub 仓库根目录放置清晰的 `LICENSE` 文件，明确第三方引用规则。
- [ ] **域名品牌保护**：至少注册 `.com` 和 `.ai` 核心域名，防止流量被截流。
- [ ] **外包协议规范**：如果雇佣自由职业者开发代码或设计 Logo，务必签署书面协议，明确 IP 归你所有。
- [ ] **防御性存证**：利用区块链时间戳或代码托管记录，证明你的创作时间点。

---

## 💡 选型建议
1. **构建开源 SaaS 但怕大厂抄袭**：首选 **AGPL-3.0** 协议，要求派生项目也必须开源。
2. **纯粹的技术分享/工具库**：选 **MIT** 或 **Apache 2.0**，最大化传播度。
3. **保护独特的 AI 提示词 (Prompts)**：虽然法律边界模糊，但可通过封装为 API 并声明 **商业秘密** 来保护。
4. **全球化产品**：优先申请 **美国商标** 与 **中国商标**，这是目前最重要的两个市场。
